# デザインパターンの説明

## Abstract Factory
### パターンの説明
- インスタンスの生成を専門に行うクラスを用意することで、整合性を必要とされる一連のオブジェクト群を間違いなく生成するためのパターン

### メリット
- 実行するメインプログラムを修正することなく、サブクラス群を環境変化に伴って交換することができる

### 利用例
- SQL文を、DBMSのパッケージによって変更したい場合
- OSによって、一部のクラスの作りを変更したい場合
